About

So, what is it?

Highly experimental game framework built on WebGL and WebSockets.

Why this project?

  • I wanted to build something for the web
  • I have never used WebGL
  • I have never built a (somewhat real) game before
    • ... particularly multiplayer

Approach

  1. Draw a scene in WebGL in 2D
  2. Send messages between client and server in 2D

Goals

  • Build with WebGL
    • accessible to anyone with a browser that supports WebGL
    • (basically everyone)
  • 3D Scene
  • Navigation
  • Multiplayer Interaction
  • Easily modifiable open source base code
  • Platform for studies that would benefit from a more diverse audience

Structure

Client-Server Model

  • Central server
  • Each client connects to this server
  • Server relays messages between clients
